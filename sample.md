# The Core of Object-Oriented Programming

Object-oriented programming (OOP) is a paradigm that organizes software design around data, or objects, rather than functions and logic.

## 1. Encapsulation
Encapsulation binds together the data and functions that manipulate the data, and that keeps both safe from outside interference and misuse. It hides the internal state of one object from the rest of the application.

## 2. Abstraction
Abstraction means hiding complex implementation details and showing only the necessary features of the object. 

## 3. Inheritance
Inheritance allows classes to inherit fields and methods from other classes. This promotes code reusability.

## 4. Polymorphism
Polymorphism allows objects of different classes to be treated as objects of a common superclass. It means "many forms" in Greek.
